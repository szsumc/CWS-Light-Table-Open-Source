# Contributing

Thanks for your interest in improving CWS Light Table.

## Before Opening A Change

- read the current [README.md](README.md)
- check the relevant files under [docs/](docs/)
- keep the project focused on its lightweight single-sheet scope

## Contribution Guidelines

- prefer small, reviewable changes
- avoid adding workbook-formatting, drawing, or multi-sheet features unless the
  project scope is explicitly updated
- keep naming based on feature responsibility rather than development-phase
  history
- preserve standalone saved-file parity for user-facing functionality
- update documentation when user-visible behavior changes

## Pull Request Checklist

- explain the user-facing purpose of the change
- note any file-format or saved-file behavior changes
- mention whether `src/core/standalone-template.js` was refreshed
- keep unrelated cleanup out of the same change when possible
